# 最小代理（Minimal Proxy）详解 - EIP-1167

## 📚 什么是最小代理？

**最小代理（Minimal Proxy）**，也叫 **克隆合约（Clone Contract）**，是一种极其节省 gas 的合约部署模式。

### 核心思想

```
不要重复部署相同的合约代码，而是：
1. 部署一次"实现合约"（包含所有逻辑）
2. 部署多个"代理合约"（只有 45 字节，指向实现合约）
3. 代理合约将所有调用转发给实现合约
```

## 🎯 为什么需要最小代理？

### 问题：传统部署成本高

```solidity
// 一个简单的 NFT 合约
contract MyNFT {
    string public name;
    string public symbol;
    address public owner;

    // ... 100+ 行逻辑代码

    function mint(address to, uint256 tokenId) public {
        // 铸造逻辑
    }
}

// 用户 A 部署自己的 NFT
MyNFT nftA = new MyNFT("CoolNFT", "COOL");
// 部署成本：~2,000,000 gas ≈ $40 (gas price: 50 gwei, ETH: $2000)

// 用户 B 部署自己的 NFT
MyNFT nftB = new MyNFT("AwesomeNFT", "AWE");
// 部署成本：~2,000,000 gas ≈ $40

// 用户 C, D, E, F... 每个人都要花 $40+
```

**问题：**
- ❌ 每个用户都要部署完整的合约代码
- ❌ 代码完全相同，只是参数不同
- ❌ 浪费大量 gas 和区块链存储空间

### 解决：使用最小代理

```solidity
// 1. 只部署一次实现合约
MyNFT implementation = new MyNFT();
// 成本：~2,000,000 gas

// 2. 为每个用户部署最小代理（45 字节）
MyNFT proxyA = Clones.clone(implementation);
// 成本：~50,000 gas ≈ $1

MyNFT proxyB = Clones.clone(implementation);
// 成本：~50,000 gas ≈ $1

MyNFT proxyC = Clones.clone(implementation);
// 成本：~50,000 gas ≈ $1

// 节省：40x 的 gas！
```

## 🔍 最小代理的工作原理

### 代理合约的字节码

```
最小代理的完整字节码只有 45 字节：

0x363d3d373d3d3d363d73[实现合约地址]5af43d82803e903d91602b57fd5bf3

拆解：
┌────────────────────────────┬──────────────────────────────┬────────────────────────────┐
│ 复制调用数据               │ 委托调用实现合约             │ 返回结果                   │
│ (10 字节)                  │ (20 字节地址 + 5 字节)       │ (10 字节)                  │
└────────────────────────────┴──────────────────────────────┴────────────────────────────┘

总计：10 + 20 + 5 + 10 = 45 字节
```

### 字节码详细分析

```assembly
// 前半部分：复制调用数据到内存
363d3d373d3d3d363d73

36        CALLDATASIZE      // 获取调用数据大小
3d        RETURNDATASIZE    // 推入 0（作为内存位置）
3d        RETURNDATASIZE    // 再次推入 0（作为 calldatasize 的位置）
37        CALLDATACOPY      // 复制调用数据到内存 [0, calldatasize)
3d        RETURNDATASIZE    // 推入 0（作为返回数据大小）
3d        RETURNDATASIZE    // 推入 0（作为返回数据位置）
3d        RETURNDATASIZE    // 推入 0（作为调用数据位置）
36        CALLDATASIZE      // 获取调用数据大小
3d        RETURNDATASIZE    // 推入 0（作为 value）
73        PUSH20            // 推入接下来的 20 字节

// 中间部分：实现合约地址（20 字节）
[实现合约地址]               // 例如：0x1234567890123456789012345678901234567890

// 后半部分：委托调用并返回结果
5af43d82803e903d91602b57fd5bf3

5a        GAS               // 获取剩余 gas
f4        DELEGATECALL      // 委托调用实现合约
3d        RETURNDATASIZE    // 获取返回数据大小
82        DUP3              // 复制
80        DUP1              // 复制
3e        RETURNDATACOPY    // 复制返回数据
90        SWAP1             // 交换栈顶元素
3d        RETURNDATASIZE    // 获取返回数据大小
91        SWAP2             // 交换
60 2b     PUSH1 0x2b        // 推入 0x2b (43)
57        JUMPI             // 如果调用成功，跳转
fd        REVERT            // 调用失败，回滚
5b        JUMPDEST          // 跳转目标
f3        RETURN            // 返回结果
```

### 执行流程

```
用户调用代理合约
    ↓
代理合约接收调用数据
    ↓
使用 DELEGATECALL 调用实现合约
    ↓
实现合约在代理合约的上下文中执行
    ↓
返回结果给用户
```

### DELEGATECALL 的关键特性

```solidity
// 代理合约
contract Proxy {
    address public owner;  // 存储槽 0
    uint256 public value;  // 存储槽 1

    function execute(address impl, bytes memory data) public {
        impl.delegatecall(data);
        // ↑ 在 Proxy 的上下文中执行 impl 的代码
    }
}

// 实现合约
contract Implementation {
    address public owner;  // 存储槽 0
    uint256 public value;  // 存储槽 1

    function setValue(uint256 _value) public {
        owner = msg.sender;  // 修改的是 Proxy 的 owner（存储槽 0）
        value = _value;      // 修改的是 Proxy 的 value（存储槽 1）
    }
}

// 关键：
// - msg.sender 保持不变（是原始调用者）
// - address(this) 是代理合约地址
// - 修改的存储是代理合约的存储
```

## 💡 完整实现示例

### 方式 1：OpenZeppelin Clones（推荐）

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/proxy/Clones.sol";

// 实现合约
contract NFTImplementation {
    string public name;
    string public symbol;
    address public owner;
    bool private initialized;

    // 不使用构造函数！使用初始化函数
    function initialize(string memory _name, string memory _symbol, address _owner) public {
        require(!initialized, "Already initialized");
        initialized = true;

        name = _name;
        symbol = _symbol;
        owner = _owner;
    }

    function mint(address to, uint256 tokenId) public {
        require(msg.sender == owner, "Not owner");
        // 铸造逻辑
    }
}

// 工厂合约
contract NFTFactory {
    address public implementation;

    event NFTCreated(address indexed nft, address indexed owner);

    constructor() {
        // 部署实现合约（只部署一次）
        implementation = address(new NFTImplementation());
    }

    // 为用户创建 NFT（使用最小代理）
    function createNFT(string memory name, string memory symbol) public returns (address) {
        // 使用 OpenZeppelin Clones 库
        address clone = Clones.clone(implementation);

        // 初始化代理合约
        NFTImplementation(clone).initialize(name, symbol, msg.sender);

        emit NFTCreated(clone, msg.sender);
        return clone;
    }

    // 使用 CREATE2 创建确定性地址的代理
    function createNFTDeterministic(
        string memory name,
        string memory symbol,
        bytes32 salt
    ) public returns (address) {
        address clone = Clones.cloneDeterministic(implementation, salt);
        NFTImplementation(clone).initialize(name, symbol, msg.sender);

        emit NFTCreated(clone, msg.sender);
        return clone;
    }

    // 预测确定性地址
    function predictAddress(bytes32 salt) public view returns (address) {
        return Clones.predictDeterministicAddress(implementation, salt);
    }
}
```

### 方式 2：手动实现

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract ManualCloneFactory {
    // 手动创建最小代理
    function clone(address implementation) public returns (address instance) {
        assembly {
            // 分配内存
            let ptr := mload(0x40)

            // 写入最小代理字节码的前半部分
            mstore(ptr, 0x3d602d80600a3d3981f3363d3d373d3d3d363d73000000000000000000000000)

            // 写入实现合约地址（20 字节）
            mstore(add(ptr, 0x14), shl(0x60, implementation))

            // 写入最小代理字节码的后半部分
            mstore(add(ptr, 0x28), 0x5af43d82803e903d91602b57fd5bf30000000000000000000000000000000000)

            // 使用 CREATE 部署
            instance := create(0, ptr, 0x37)
        }

        require(instance != address(0), "Clone creation failed");
    }

    // 使用 CREATE2 创建确定性地址的代理
    function cloneDeterministic(address implementation, bytes32 salt) public returns (address instance) {
        assembly {
            let ptr := mload(0x40)

            mstore(ptr, 0x3d602d80600a3d3981f3363d3d373d3d3d363d73000000000000000000000000)
            mstore(add(ptr, 0x14), shl(0x60, implementation))
            mstore(add(ptr, 0x28), 0x5af43d82803e903d91602b57fd5bf30000000000000000000000000000000000)

            // 使用 CREATE2 部署
            instance := create2(0, ptr, 0x37, salt)
        }

        require(instance != address(0), "Clone creation failed");
    }

    // 预测 CREATE2 地址
    function predictDeterministicAddress(
        address implementation,
        bytes32 salt,
        address deployer
    ) public pure returns (address predicted) {
        assembly {
            let ptr := mload(0x40)

            // 构建字节码
            mstore(ptr, 0x3d602d80600a3d3981f3363d3d373d3d3d363d73000000000000000000000000)
            mstore(add(ptr, 0x14), shl(0x60, implementation))
            mstore(add(ptr, 0x28), 0x5af43d82803e903d91602b57fd5bf30000000000000000000000000000000000)

            // 计算字节码哈希
            let hash := keccak256(ptr, 0x37)

            // 计算 CREATE2 地址
            mstore(0x00, 0xff)
            mstore(0x01, shl(0x60, deployer))
            mstore(0x15, salt)
            mstore(0x35, hash)

            predicted := keccak256(0x00, 0x55)
        }
    }
}
```

### TypeScript 使用示例

```typescript
import { ethers } from "hardhat";

async function main() {
  const [deployer, user1, user2] = await ethers.getSigners();

  // 1. 部署工厂合约
  const NFTFactory = await ethers.getContractFactory("NFTFactory");
  const factory = await NFTFactory.deploy();
  await factory.waitForDeployment();

  console.log("Factory deployed to:", await factory.getAddress());
  console.log("Implementation:", await factory.implementation());

  // 2. 用户 1 创建 NFT（使用最小代理）
  const tx1 = await factory.connect(user1).createNFT("CoolNFT", "COOL");
  const receipt1 = await tx1.wait();

  // 从事件中获取代理地址
  const event1 = receipt1.logs.find(log => {
    try {
      return factory.interface.parseLog(log)?.name === "NFTCreated";
    } catch {
      return false;
    }
  });
  const nft1Address = factory.interface.parseLog(event1!).args.nft;

  console.log("\nUser 1's NFT deployed to:", nft1Address);
  console.log("Gas used:", receipt1.gasUsed.toString());

  // 3. 用户 2 创建 NFT
  const tx2 = await factory.connect(user2).createNFT("AwesomeNFT", "AWE");
  const receipt2 = await tx2.wait();

  const event2 = receipt2.logs.find(log => {
    try {
      return factory.interface.parseLog(log)?.name === "NFTCreated";
    } catch {
      return false;
    }
  });
  const nft2Address = factory.interface.parseLog(event2!).args.nft;

  console.log("\nUser 2's NFT deployed to:", nft2Address);
  console.log("Gas used:", receipt2.gasUsed.toString());

  // 4. 获取代理合约实例
  const nft1 = await ethers.getContractAt("NFTImplementation", nft1Address);
  const nft2 = await ethers.getContractAt("NFTImplementation", nft2Address);

  // 5. 验证数据
  console.log("\nNFT 1:");
  console.log("  Name:", await nft1.name());
  console.log("  Symbol:", await nft1.symbol());
  console.log("  Owner:", await nft1.owner());

  console.log("\nNFT 2:");
  console.log("  Name:", await nft2.name());
  console.log("  Symbol:", await nft2.symbol());
  console.log("  Owner:", await nft2.owner());

  // 6. 使用确定性地址（CREATE2）
  const salt = ethers.id("my-nft-v1");
  const predictedAddress = await factory.predictAddress(salt);

  console.log("\nPredicted address:", predictedAddress);

  const tx3 = await factory.createNFTDeterministic("DeterministicNFT", "DET", salt);
  await tx3.wait();

  const nft3 = await ethers.getContractAt("NFTImplementation", predictedAddress);
  console.log("Actual address:", await nft3.getAddress());
  console.log("Addresses match:", predictedAddress === await nft3.getAddress());
}

main().catch(console.error);
```

## 📊 成本对比

### Gas 成本对比

```
传统部署（每次部署完整合约）:
├─ 部署 User1 的 NFT: ~2,000,000 gas
├─ 部署 User2 的 NFT: ~2,000,000 gas
├─ 部署 User3 的 NFT: ~2,000,000 gas
└─ 总计 3 个用户:     ~6,000,000 gas

使用最小代理:
├─ 部署实现合约（一次）: ~2,000,000 gas
├─ 部署工厂合约（一次）: ~200,000 gas
├─ User1 创建代理:       ~50,000 gas
├─ User2 创建代理:       ~50,000 gas
├─ User3 创建代理:       ~50,000 gas
└─ 总计 3 个用户:        ~2,350,000 gas

节省: 60%+

如果有 100 个用户:
传统方式: 200,000,000 gas
最小代理:   7,200,000 gas
节省: 96%+
```

### 字节码大小对比

```
完整合约:
├─ 字节码: ~10,000 bytes
└─ 成本: 存储 10KB 到链上

最小代理:
├─ 字节码: 45 bytes
└─ 成本: 只存储 45 字节
```

## ⚠️ 注意事项

### 1. 不能使用构造函数

```solidity
// ❌ 错误：不能使用构造函数
contract BadImplementation {
    string public name;

    constructor(string memory _name) {
        name = _name;  // 这会在实现合约部署时执行，不是在代理中
    }
}

// ✅ 正确：使用初始化函数
contract GoodImplementation {
    string public name;
    bool private initialized;

    function initialize(string memory _name) public {
        require(!initialized, "Already initialized");
        initialized = true;
        name = _name;
    }
}
```

### 2. 防止重复初始化

```solidity
// ❌ 危险：可以多次初始化
function initialize(string memory _name) public {
    name = _name;
}

// ✅ 安全：使用初始化标志
bool private initialized;

function initialize(string memory _name) public {
    require(!initialized, "Already initialized");
    initialized = true;
    name = _name;
}

// ✅ 更安全：使用 OpenZeppelin Initializable
import "@openzeppelin/contracts-upgradeable/proxy/utils/Initializable.sol";

contract MyContract is Initializable {
    function initialize(string memory _name) public initializer {
        name = _name;
    }
}
```

### 3. 存储布局必须一致

```solidity
// 实现合约
contract Implementation {
    address public owner;   // 存储槽 0
    uint256 public value;   // 存储槽 1
    string public name;     // 存储槽 2
}

// ❌ 错误：代理合约的存储布局不同
contract Proxy {
    uint256 public value;   // 存储槽 0 ← 位置错误！
    address public owner;   // 存储槽 1 ← 位置错误！
}

// ✅ 正确：完全相同的存储布局
contract Proxy {
    address public owner;   // 存储槽 0 ← 正确
    uint256 public value;   // 存储槽 1 ← 正确
    string public name;     // 存储槽 2 ← 正确
}
```

### 4. 调用成本略高

```
直接调用完整合约:
└─ Gas: 基础调用成本

调用最小代理:
├─ 代理合约开销: ~2,600 gas（DELEGATECALL + 数据复制）
└─ 实现合约执行: 基础调用成本

总计: 约增加 2,600 gas per call

权衡: 部署节省 95% gas vs 调用增加 2,600 gas
对于大多数应用，这是值得的
```

### 可升级合约（相关概念）

```
最小代理 vs 可升级代理:

最小代理（EIP-1167）:
├─ 代理地址固定
├─ 实现合约固定（不能升级）
├─ 极低成本（45 字节）
└─ 适用于大量部署

可升级代理（EIP-1967）:
├─ 代理地址固定
├─ 实现合约可更换（可升级）
├─ 成本较高（~200+ 字节）
└─ 适用于需要升级的合约
```

## 🎓 总结

### 什么是最小代理？

一个只有 **45 字节** 的极简合约，它将所有调用 **委托** 给实现合约执行。

### 为什么使用最小代理？

| 优势 | 说明 |
|------|------|
| **极低成本** | 部署成本降低 95%+ |
| **节省空间** | 只存储 45 字节而非数千字节 |
| **代码重用** | 一份实现，无限代理 |
| **灵活性** | 每个代理有独立状态 |

### 核心原理

```
用户 → 代理合约（45字节）→ DELEGATECALL → 实现合约
                ↑                               ↓
                └─────── 在代理上下文执行 ←───────┘
```

### 关键要点

1. ✅ 使用初始化函数，不用构造函数
2. ✅ 防止重复初始化
3. ✅ 保持存储布局一致
4. ✅ 接受略微增加的调用成本
5. ✅ 适合需要大量部署相同逻辑的场景
