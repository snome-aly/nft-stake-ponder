# 合约部署原理详解

## ❓ 问题

在部署脚本中，只看到这样一行代码：

```typescript
await deploy("YourContract", {
  from: deployer,
  args: [deployer],
  log: true,
});
```

这个 `deploy()` 函数是怎么把合约部署到链上的呢？

## ✅ 答案：完整的编译、签名、发送流程

`deploy()` 函数内部执行了一系列复杂的操作，将 Solidity 源码最终部署到区块链上。

## 🔄 完整部署流程

### 阶段 1：编译合约（获取字节码）

```
YourContract.sol (Solidity 源码)
    ↓ Solidity 编译器
contracts/YourContract.sol:
    ↓
bytecode:       0x608060405234801561001057600080fd5b506040516...
deployedBytecode: 0x608060405234801561001057600080fd5b50600436...
ABI:            [{"type":"constructor","inputs":[{"name":"_owner",...}]
```

#### 1.1 什么是 bytecode？

```solidity
// YourContract.sol
contract YourContract {
    address public immutable owner;
    string public greeting = "Building Unstoppable Apps!!!";

    constructor(address _owner) {
        owner = _owner;
    }

    function setGreeting(string memory _newGreeting) public payable {
        greeting = _newGreeting;
    }
}
```

编译后生成：

```javascript
// bytecode（创建字节码）
// 包含：构造函数逻辑 + 部署代码 + 运行时字节码
const bytecode = "0x608060405234801561001057600080fd5b50604051610...";

// deployedBytecode（运行时字节码）
// 部署到链上后实际存储的代码，不包含构造函数
const deployedBytecode = "0x608060405234801561001057600080fd5b50600436...";

// ABI（应用程序二进制接口）
// 描述合约的函数、事件、参数类型
const abi = [
  {
    "type": "constructor",
    "inputs": [{"name": "_owner", "type": "address"}]
  },
  {
    "type": "function",
    "name": "setGreeting",
    "inputs": [{"name": "_newGreeting", "type": "string"}],
    "outputs": []
  },
  // ...
];
```

### 阶段 2：编码构造函数参数

```typescript
// 部署脚本中指定的参数
await deploy("YourContract", {
  args: [deployer],  // deployer = "0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266"
});
```

#### 2.1 参数编码过程

```javascript
// 1. 获取构造函数的 ABI
const constructorAbi = {
  type: "constructor",
  inputs: [{ name: "_owner", type: "address" }]
};

// 2. 编码参数（使用 ABI 编码规则）
const encodedArgs = ethers.AbiCoder.defaultAbiCoder().encode(
  ["address"],                                      // 参数类型
  ["0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266"]  // 参数值
);
// 结果：0x000000000000000000000000f39fd6e51aad88f6f4ce6ab8827279cfffb92266

// 3. 拼接字节码和编码后的参数
const deployData = bytecode + encodedArgs.slice(2);
// slice(2) 去掉 0x 前缀
```

### 阶段 3：构造部署交易

```typescript
// deploy() 函数内部构造的交易对象
const deployTx = {
  from: "0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266",  // 部署者地址
  data: "0x608060405234801561001057...000000000000000000000000f39fd6e51aad88f6f4ce6ab8827279cfffb92266",  // 字节码 + 参数
  gas: 500000,        // Gas 限制（自动估算或手动设置）
  gasPrice: "10000000000",  // Gas 价格（10 Gwei）
  nonce: 0,           // 账户的交易序号
  chainId: 11155111,  // Sepolia 链 ID
  // 注意：部署交易没有 'to' 字段！
};
```

#### 3.1 为什么部署交易没有 `to` 字段？

```javascript
// 普通交易（调用合约函数）
{
  from: "0xf39F...",
  to: "0x5FbDB...",    // ← 目标合约地址
  data: "0x...",       // 函数调用数据
}

// 部署交易（创建新合约）
{
  from: "0xf39F...",
  to: null,            // ← 没有目标地址！
  data: "0x...",       // 合约字节码 + 构造函数参数
}
// 当 to 为 null 时，EVM 知道这是创建合约的交易
```

### 阶段 4：签名交易

```typescript
// 1. 从私钥创建钱包（签名器）
const wallet = new ethers.Wallet(deployerPrivateKey, provider);

// 2. 签名交易
const signedTx = await wallet.signTransaction(deployTx);

// 签名过程（ECDSA 椭圆曲线数字签名算法）
// 1. 计算交易哈希
const txHash = keccak256(rlpEncode(deployTx));

// 2. 使用私钥签名
const signature = sign(txHash, privateKey);

// 3. 添加签名字段
const signedTransaction = {
  ...deployTx,
  v: signature.v,  // 恢复参数
  r: signature.r,  // 签名 r 值
  s: signature.s,  // 签名 s 值
};
```

#### 4.1 为什么需要签名？

```
签名的作用：
1. 证明你拥有私钥（身份认证）
2. 证明交易内容未被篡改（完整性）
3. 防止重放攻击（通过 nonce）

没有私钥 = 无法签名 = 无法发送交易 = 无法部署合约
```

### 阶段 5：发送交易到区块链

```typescript
// 1. 连接到区块链节点（通过 RPC）
const provider = new ethers.JsonRpcProvider(
  "https://eth-sepolia.g.alchemy.com/v2/your-api-key"
);

// 2. 发送签名后的交易
const txResponse = await provider.broadcastTransaction(signedTx);

// 3. 等待交易被矿工打包（确认）
const receipt = await txResponse.wait();
```

#### 5.1 交易在网络中的传播

```
你的电脑
    ↓ 发送交易
Alchemy RPC 节点
    ↓ 广播
Ethereum P2P 网络
    ↓ 传播
矿工/验证者节点
    ↓ 选择交易打包
新区块
    ↓ 广播
全网节点确认
```

### 阶段 6：EVM 执行合约创建

```
1. 验证者收到交易
    ↓
2. 验证签名和余额
    ↓
3. 执行交易：
   - 扣除 Gas 费用
   - 运行 bytecode 中的构造函数
   - 存储运行时字节码到新地址
    ↓
4. 生成合约地址（使用 CREATE 操作码）
   合约地址 = keccak256(deployer_address + nonce)[12:]
    ↓
5. 返回交易回执
```

#### 6.1 合约地址是如何计算的？

```javascript
// 方法 1：CREATE（标准部署）
const contractAddress = ethers.getCreateAddress({
  from: "0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266",  // 部署者地址
  nonce: 0  // 部署者的 nonce
});
// 计算公式：keccak256(rlp([deployer_address, nonce]))[12:]
// 结果：0x5FbDB2315678afecb367f032d93F642f64180aa3

// 方法 2：CREATE2（确定性部署）
const contractAddress = ethers.getCreate2Address(
  factoryAddress,  // 工厂合约地址
  salt,            // 盐值（任意 32 字节）
  bytecodeHash     // 字节码的 keccak256 哈希
);
// 计算公式：keccak256(0xff + factory + salt + keccak256(bytecode))[12:]
```

### 阶段 7：保存部署信息

```typescript
// hardhat-deploy 插件保存部署数据
const deploymentData = {
  address: receipt.contractAddress,  // 0x5FbDB2315678afecb367f032d93F642f64180aa3

    blockNumber: receipt.blockNumber,
    gasUsed: receipt.gasUsed.toString(),

  JSON.stringify(deploymentData, null, 2)
);
```

## 💡 deploy() 函数的内部实现

### 简化版实现

```typescript
// hardhat-deploy 插件的 deploy() 函数（简化版）
async function deploy(contractName: string, options: DeployOptions) {
  // 1. 读取编译产物
  const artifact = await hre.artifacts.readArtifact(contractName);
  const { abi, bytecode } = artifact;

  // 2. 编码构造函数参数
  const encodedArgs = ethers.AbiCoder.defaultAbiCoder().encode(
    getConstructorTypes(abi),
    options.args
  );

  // 3. 拼接字节码和参数
  const deployData = bytecode + encodedArgs.slice(2);

  // 4. 创建部署交易
  const deployTx = {
    from: options.from,
    data: deployData,
    gasLimit: options.gasLimit || (await estimateGas()),
  };

  // 5. 获取签名器（钱包）
  const signer = await hre.ethers.getSigner(options.from);

  // 6. 发送交易
  const txResponse = await signer.sendTransaction(deployTx);

  // 7. 等待确认
  const receipt = await txResponse.wait();

  // 8. 保存部署信息
  await saveDeployment(contractName, {
    address: receipt.contractAddress,
    abi,
    receipt,
    args: options.args,
  });

  if (options.log) {
    console.log(`✅ ${contractName} deployed at ${receipt.contractAddress}`);
  }

  return {
    address: receipt.contractAddress,
    receipt,
    // ...
  };
}
```

## 🎓 关键技术概念

### 1. 字节码（Bytecode）

```
什么是字节码？
- EVM（以太坊虚拟机）的机器码
- 16 进制格式，以 0x 开头
- 包含操作码（OPCODES）和数据

示例：
0x6080604052  →  PUSH1 80 PUSH1 40 MSTORE
↑ PUSH1       →  将 1 字节数据推入栈
  6080        →  数据：0x80
  ↑ PUSH1     →  将 1 字节数据推入栈
    604052    →  数据：0x40
    ↑ MSTORE  →  将数据存储到内存
```

### 2. ABI 编码

```typescript
// ABI 编码规则（简化版）
// 每个参数占 32 字节（256 位）

// 示例 1：编码一个 address
const address = "0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266";
const encoded = "0x000000000000000000000000f39fd6e51aad88f6f4ce6ab8827279cfffb92266";
//                 ↑ 填充到 32 字节

// 示例 2：编码多个参数
// function transfer(address to, uint256 amount)
const encoded = ethers.AbiCoder.defaultAbiCoder().encode(
  ["address", "uint256"],
  ["0xf39F...", 1000]
);
// 结果：
// 0x000000000000000000000000f39fd6e51aad88f6f4ce6ab8827279cfffb92266  ← to
//   00000000000000000000000000000000000000000000000000000000000003e8  ← amount (1000)
```

### 3. Gas 和 Gas Price

```typescript
// Gas：执行操作所需的计算单位
const gasLimit = 500000;  // 最多使用 500,000 Gas

// Gas Price：每单位 Gas 的价格（以 wei 计）
const gasPrice = ethers.parseUnits("10", "gwei");  // 10 Gwei

// 交易费用计算
const txCost = gasUsed * gasPrice;
// 例如：200,000 Gas × 10 Gwei = 2,000,000 Gwei = 0.002 ETH

// 部署合约通常消耗的 Gas：
// - 简单合约：200,000 - 500,000 Gas
// - 中等合约：500,000 - 1,000,000 Gas
// - 复杂合约：1,000,000 - 3,000,000 Gas
```

### 4. 交易回执（Transaction Receipt）

```typescript
const receipt = {
  transactionHash: "0x123...",      // 交易哈希
  blockNumber: 12345678,            // 区块号
  blockHash: "0xabc...",            // 区块哈希
  from: "0xf39F...",                // 发送者
  to: null,                         // 部署交易 to 为 null
  contractAddress: "0x5FbDB...",    // 新部署的合约地址
  gasUsed: BigInt(200000),          // 实际消耗的 Gas
  status: 1,                        // 1=成功, 0=失败
  logs: [                           // 事件日志
    {
      address: "0x5FbDB...",
      topics: ["0x..."],
      data: "0x..."
    }
  ]
};
```

## 🎯 总结

### deploy() 函数做了什么？

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1️⃣ | 读取编译产物 | 获取 bytecode、ABI |
| 2️⃣ | 编码构造函数参数 | 使用 ABI 编码规则 |
| 3️⃣ | 拼接字节码和参数 | bytecode + encodedArgs |
| 4️⃣ | 构造部署交易 | from + data + gas |
| 5️⃣ | 签名交易 | 使用私钥签名 |
| 6️⃣ | 发送交易 | 通过 RPC 节点 |
| 7️⃣ | 等待确认 | 矿工打包到区块 |
| 8️⃣ | EVM 执行 | 运行构造函数，存储字节码 |
| 9️⃣ | 计算合约地址 | keccak256(deployer + nonce) |
| 🔟 | 保存部署信息 | 写入 JSON 文件 |

### 核心原理

1. **合约部署 = 特殊的交易**
   - to 字段为 null
   - data 字段包含字节码

2. **字节码 = 合约的机器码**
   - 由 Solidity 编译器生成
   - 存储在区块链上

3. **构造函数只执行一次**
   - 部署时运行
   - 不会存储到链上
   - 用于初始化状态变量

4. **合约地址是确定的**
   - 由部署者地址和 nonce 计算
   - 可以在部署前预测

5. **需要支付 Gas 费用**
   - 部署合约消耗大量 Gas
   - 费用 = gasUsed × gasPrice
