## 什么是并行路由

**定义：** 在同一个 Layout 中同时渲染多个页面（slots），每个页面可以独立导航。

**核心特点：**
- 使用 `@folder` 语法定义插槽（slot）
- 每个插槽独立渲染，互不影响
- 可以有独立的加载状态和错误处理
- 适合仪表板、模态框、分栏布局等场景

### 简单示例

```
app/
├── layout.tsx
├── @team/
│   └── page.tsx
├── @analytics/
│   └── page.tsx
└── page.tsx
```

**Layout 中使用：**

```typescript
// app/layout.tsx
export default function Layout({
  children,
  team,      // @team 插槽
  analytics, // @analytics 插槽
}) {
  return (
    <div>
      <div>{children}</div>
      <div>{team}</div>
      <div>{analytics}</div>
    </div>
  )
}
```

**渲染结果：**

```html
<div>
  <div><!-- children: app/page.tsx --></div>
  <div><!-- team: app/@team/page.tsx --></div>
  <div><!-- analytics: app/@analytics/page.tsx --></div>
</div>
```

## 基础语法

### 文件结构

```
app/
├── layout.tsx              # Layout 接收所有插槽作为 props
├── page.tsx                # 默认内容（children）
├── @team/                  # team 插槽
│   ├── page.tsx            # /team 路由的内容
│   ├── settings/
│   │   └── page.tsx        # /team/settings 的内容
│   └── default.tsx         # 默认后备内容
├── @analytics/             # analytics 插槽
│   ├── page.tsx
│   └── default.tsx
└── default.tsx             # children 的默认内容
```

### Layout 定义

```typescript
// app/layout.tsx
export default function DashboardLayout({
  children,    // 默认插槽
  team,        // @team 插槽
  analytics,   // @analytics 插槽
}: {
  children: React.ReactNode
  team: React.ReactNode
  analytics: React.ReactNode
}) {
  return (
    <div className="dashboard">
      {/* 主内容区 */}
      <main>{children}</main>

      {/* 团队面板 */}
      <aside className="team-panel">{team}</aside>

      {/* 分析面板 */}
      <aside className="analytics-panel">{analytics}</aside>
    </div>
  )
}
```

```typescript

export default function DashboardPage() {

    </div>
```

## 路由跳转处理

### 关键问题：跳转时插槽会发生什么？

**核心规则：**
1. **匹配的插槽** - 如果插槽有对应路由，渲染该路由
2. **不匹配的插槽** - 保持当前活动状态（Active State）
3. **完全不匹配** - 使用 `default.tsx`（如果有）或渲染 404

### 场景 1: 插槽有匹配路由

**文件结构：**
```
app/
├── layout.tsx
├── @team/
│   ├── page.tsx          # 匹配 /
│   └── settings/
│       └── page.tsx      # 匹配 /settings
└── @analytics/
    └── page.tsx          # 只匹配 /
```

**跳转行为：**

```typescript
// 当前 URL: /
// 渲染:
// - @team/page.tsx
// - @analytics/page.tsx

// 跳转到 /settings
<Link href="/settings">Settings</Link>

// 新的渲染:
// - @team/settings/page.tsx  ✅ 有匹配，渲染新内容
// - @analytics/page.tsx      ⚠️ 无匹配，保持旧内容
```

**实际代码：**

```typescript
// app/@team/page.tsx
export default function TeamPage() {
  return (
    <div>
      <h2>团队列表</h2>
      <Link href="/settings">团队设置</Link>
    </div>
  )
}

// app/@team/settings/page.tsx
export default function TeamSettingsPage() {
  return (
    <div>
      <h2>团队设置</h2>
      <Link href="/">返回</Link>
    </div>
  )
}

// app/@analytics/page.tsx
export default function AnalyticsPage() {
  return <h2>分析数据</h2>
}
```

**点击 "团队设置" 后：**
```
URL: /settings

渲染:
├─ @team: TeamSettingsPage     ← 变化了
└─ @analytics: AnalyticsPage   ← 保持不变（因为没有 @analytics/settings/page.tsx）
```

### 场景 2: 使用 default.tsx

**文件结构：**
```
app/
├── layout.tsx
├── @team/
│   ├── page.tsx
│   ├── settings/
│   │   └── page.tsx
│   └── default.tsx        # ← 默认后备内容
└── @analytics/
    ├── page.tsx
    └── default.tsx        # ← 默认后备内容
```

**default.tsx 的作用：**

当插槽没有匹配的路由时，渲染 `default.tsx`。

```typescript
// app/@analytics/default.tsx
export default function AnalyticsDefault() {
  return (
    <div>
      <p>当前路由没有分析数据</p>
      <Link href="/">返回首页查看分析</Link>
    </div>
  )
}
```

**跳转行为：**

```typescript
// URL: /
// 渲染: @analytics/page.tsx

// 跳转到 /settings
// 渲染: @analytics/default.tsx  ← 因为没有 @analytics/settings/page.tsx
```

### 场景 3: 没有 default.tsx

**如果没有 default.tsx 会怎样？**

```
app/
├── layout.tsx
├── @team/
│   ├── page.tsx
│   └── settings/
│       └── page.tsx
└── @analytics/
    └── page.tsx           # 只有 page.tsx，没有 default.tsx
```

**跳转行为：**

```typescript
// URL: /
// 渲染: @analytics/page.tsx  ✅

// 软导航（客户端导航）到 /settings
<Link href="/settings">Settings</Link>
// 渲染: @analytics/page.tsx  ✅ 保持活动状态

// 硬导航（刷新页面）到 /settings
// 渲染: 404 错误 ❌ 因为没有 @analytics/settings/page.tsx 也没有 default.tsx
```

**重要：** 软导航 vs 硬导航

| 导航类型 | 行为 |
|---------|------|
| **软导航**（客户端） | 保持活动状态（Active State） |
| **硬导航**（刷新） | 需要 default.tsx，否则 404 |

**软导航示例：**
```typescript
// 使用 Link 组件（软导航）
<Link href="/settings">Settings</Link>

// 使用 useRouter（软导航）
const router = useRouter()
router.push('/settings')
```

**硬导航示例：**
```typescript
// 直接输入 URL
// 刷新页面
// 外部链接
```

## 活动状态与默认插槽

### 活动状态（Active State）

**定义：** Next.js 会记住每个插槽当前渲染的是哪个页面，即使导航到没有该插槽定义的路由。

**示例：**

```typescript
// 1. 访问 /
// @team 渲染: @team/page.tsx

// 2. 软导航到 /settings
// @team 渲染: @team/settings/page.tsx

// 3. 软导航到 /other (没有 @team/other/page.tsx)
// @team 渲染: @team/settings/page.tsx  ← 保持上一个活动状态
```

**注意：** 只在软导航时生效，硬导航会丢失活动状态。

### default.tsx 的优先级

```
插槽匹配路由吗？
├─ 是 → 渲染对应的 page.tsx
└─ 否
    ├─ 是软导航？
    │   └─ 是 → 保持活动状态
    └─ 是硬导航？
        ├─ 有 default.tsx → 渲染 default.tsx
        └─ 无 default.tsx → 404 错误
```

### 推荐做法

**总是提供 default.tsx：**

```typescript
// app/@slot/default.tsx
export default function SlotDefault() {
  return null  // 或返回一个占位符
}
```

**原因：**
- 避免硬导航时的 404
- 提供更好的用户体验
- 可以显示有用的提示信息

### 场景 2: 模态框（Modal）

**需求：** 打开模态框时 URL 改变，但页面不刷新。

**文件结构：**
```
app/
├── layout.tsx
├── page.tsx               # 首页
├── products/
│   └── page.tsx           # 产品列表
├── @modal/
│   ├── default.tsx        # 默认不显示模态框
│   └── (..)photo/
│       └── [id]/
│           └── page.tsx   # 模态框内容
└── photo/
    └── [id]/
        └── page.tsx       # 完整页面
```

**Layout：**
```typescript
// app/layout.tsx
export default function RootLayout({
  children,
  modal,
}: {
  children: React.ReactNode
  modal: React.ReactNode
}) {
  return (
    <html>
      <body>
        {children}
        {modal}  {/* 模态框叠加在内容上方 */}
      </body>
    </html>
  )
}
```

**模态框：**
```typescript
// app/@modal/default.tsx
export default function ModalDefault() {
  return null  // 默认不显示
}

// app/@modal/(..)photo/[id]/page.tsx
'use client'
import { useRouter } from 'next/navigation'

export default function PhotoModal({ params }: { params: { id: string } }) {
  const router = useRouter()

  return (
    <div className="fixed inset-0 bg-black/50 flex items-center justify-center">
      <div className="bg-white p-8 rounded-lg">
        <button onClick={() => router.back()}>关闭</button>
        <h2>照片 #{params.id}</h2>
        <img src={`/photos/${params.id}.jpg`} alt="" />
      </div>
    </div>
  )
}
```

**产品列表：**
```typescript
// app/products/page.tsx
export default function ProductsPage() {
  return (
    <div className="grid grid-cols-3 gap-4">
      {[1, 2, 3, 4, 5, 6].map(id => (
        <Link key={id} href={`/photo/${id}`}>
          <img src={`/photos/${id}.jpg`} alt="" />
        </Link>
      ))}
    </div>
  )
}
```

**完整照片页：**
```typescript
// app/photo/[id]/page.tsx
export default function PhotoPage({ params }: { params: { id: string } }) {
  return (
    <div>
      <Link href="/products">← 返回</Link>
      <h1>照片 #{params.id}</h1>
      <img src={`/photos/${params.id}.jpg`} className="w-full" />
    </div>
  )
}
```

**用户操作：**

```typescript
// 1. 访问 /products
children: ProductsPage
@modal: null (default.tsx)

// 2. 点击照片 1（软导航）
URL: /photo/1

// 3. 点击关闭

@modal: null (default.tsx)

// 4. 直接访问 /photo/1 (硬导航，如刷新页面)

```

**关键：** 使用拦截路由 `(..)` 实现软导航显示模态框，硬导航显示完整页面。

## 与拦截路由结合

### 拦截路由（Intercepting Routes）

**语法：**
- `(.)` - 拦截同级
- `(..)` - 拦截上一级
- `(..)(..)` - 拦截上两级
- `(...)` - 拦截根目录

### 完整示例：照片画廊

**文件结构：**
```
app/
├── layout.tsx
├── page.tsx                    # 首页
├── gallery/
│   └── page.tsx                # 画廊列表
├── photo/
│   └── [id]/
│       └── page.tsx            # 完整照片页
└── @modal/
    ├── default.tsx
    └── (..)photo/
        └── [id]/
            └── page.tsx        # 拦截 /photo/[id]，显示模态框
```

**工作原理：**

```typescript
// 从 /gallery 软导航到 /photo/1
// Next.js 检查是否有拦截路由
// 找到 @modal/(..)photo/[id]/page.tsx
// 渲染 @modal 插槽，children 保持 gallery

// 从外部直接访问 /photo/1 (硬导航)
// 没有之前的导航历史
// 渲染正常的 photo/[id]/page.tsx
```

**完整代码：**

```typescript
// app/layout.tsx
export default function RootLayout({ children, modal }) {
  return (
    <html>
      <body>
        {children}
        {modal}
      </body>
    </html>
  )
}

// app/gallery/page.tsx
export default function GalleryPage() {
  const photos = [1, 2, 3, 4, 5, 6]
```

## 常见问题

### 1. 插槽没有内容时显示空白

**问题：**
```typescript
// 访问 /settings，但没有 @analytics/settings/page.tsx
// @analytics 插槽显示空白
```

**解决：**
```typescript
// 添加 app/@analytics/default.tsx
export default function AnalyticsDefault() {
  return (
    <div className="text-gray-500">
      <p>当前页面没有分析数据</p>
    </div>
  )
}
```

### 2. 刷新页面后 404

**问题：**
```typescript
// 软导航到 /settings 正常
// 刷新页面后 404
```

**原因：** 没有 default.tsx，硬导航找不到匹配路由。

**解决：** 为每个插槽添加 default.tsx。

### 3. 如何在插槽之间传递数据？

**❌ 不能直接传递 props：**
```typescript
// Layout 不能向插槽传递自定义 props
<aside>{team}</aside>  // team 是自动注入的
```

**✅ 使用 Context：**
```typescript
// app/layout.tsx
'use client'
import { DashboardProvider } from './DashboardContext'

export default function Layout({ children, team, analytics }) {
  return (
    <DashboardProvider>
      <main>{children}</main>
      <aside>{team}</aside>
      <aside>{analytics}</aside>
    </DashboardProvider>
  )
}

// app/@team/page.tsx
'use client'
import { useDashboard } from '../DashboardContext'

export default function TeamPage() {
  const { selectedDate } = useDashboard()
  return <div>团队数据 ({selectedDate})</div>
}
```

### 4. 如何条件渲染插槽？

```typescript
// app/layout.tsx
export default function Layout({
  children,
  team,
  analytics,
}: {
  children: React.ReactNode
  team: React.ReactNode
  analytics: React.ReactNode
}) {
  const showSidebar = true  // 根据某些条件

  return (
    <div>
      <main>{children}</main>
      {showSidebar && (
        <aside>
          {team}
          {analytics}
        </aside>
      )}
    </div>
  )
}
```

### 5. 如何知道当前活动的插槽？

**使用 useSelectedLayoutSegment：**

```typescript
'use client'
import { useSelectedLayoutSegment } from 'next/navigation'

export default function TeamSlot() {
  const segment = useSelectedLayoutSegment('team')
  // segment = 'settings' 当 URL 是 /settings
  // segment = null 当 URL 是 /

  return (
    <div>
      <h2>团队</h2>
      {segment === 'settings' && <p>当前在设置页</p>}
    </div>
  )
}
```

## 最佳实践

### 1. 总是提供 default.tsx

```typescript

  return null  // 或有意义的占位内容
}
```

### 2. 使用语义化的插槽名

```typescript
// ✅ 好的命名

@modal
@toolbar
@notifications

// ❌ 不好的命名
@slot1
@content2
@thing
```

### 3. 文档化插槽用途

```typescript

/**
 * Dashboard Layout
 *
 * 插槽:
 * - @sidebar: 左侧导航栏
 * - @content: 主内容区
 * - @notifications: 右上角通知
 */
export default function DashboardLayout({
  sidebar,
  content,
  notifications,

  // ...
}
```

### 4. 合理使用并行路由

**适用场景：**
- ✅ 仪表板布局
- ✅ 模态框/对话框
- ✅ 分栏布局（邮件客户端）
- ✅ 多个独立更新的区域

**不适用场景：**
- ❌ 简单的页面布局（用普通 Layout）
- ❌ 父子关系的内容（用嵌套路由）
- ❌ 条件渲染（用客户端组件）

```

├── page.tsx
├── loading.tsx       # 加载状态
├── error.tsx         # 错误状态
└── default.tsx       # 默认状态
```

## 总结

### 核心要点

**并行路由 = 在同一个 Layout 中同时渲染多个页面**

**语法：**
```typescript
app/
├── layout.tsx         // 接收插槽作为 props
├── @slot/            // @名称 定义插槽
│   ├── page.tsx
│   └── default.tsx   // 必须提供！
```

**跳转行为：**
```
插槽有匹配路由？
├─ 是 → 渲染新内容
└─ 否
    ├─ 软导航 → 保持活动状态
    └─ 硬导航 → 使用 default.tsx 或 404
```

**关键概念：**
1. **插槽（Slot）** - `@folder` 语法
2. **活动状态（Active State）** - 软导航时保持
3. **默认后备（default.tsx）** - 硬导航时使用
4. **拦截路由** - 结合使用实现模态框

### 快速参考

```typescript
// 定义插槽
app/@team/page.tsx

// Layout 中使用
export default function Layout({ team }) {
  return <div>{team}</div>
}

// 提供默认内容
app/@team/default.tsx
export default function TeamDefault() {
  return null
}

// 拦截路由实现模态框
app/@modal/(..)photo/[id]/page.tsx
```

并行路由是 Next.js App Router 的高级特性，主要用于复杂的布局场景。记住关键点：**总是提供 default.tsx，理解软导航 vs 硬导航的区别**！
