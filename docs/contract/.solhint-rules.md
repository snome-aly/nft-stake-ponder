## 📖 规则详细说明

### 基础配置

#### `"extends": "solhint:recommended"`
- **说明**：继承 solhint 推荐的规则集
- **包含**：约 30+ 个最佳实践规则
- **作用**：自动启用常见的安全和代码质量检查
- **查看完整列表**：https://github.com/protofire/solhint/blob/master/conf/rulesets/solhint-recommended.js

---

### 自定义规则

#### 1. `compiler-version`
```json
"compiler-version": ["error", "^0.8.20"]
```
- **级别**：`error`（错误）
- **作用**：检查 `pragma solidity` 版本声明
- **配置**：必须是 `^0.8.20`（0.8.20 或更高的 0.8.x）
- **原因**：
  - 确保使用正确的编译器版本
  - 避免版本不兼容问题
  - 0.8.x 有内置溢出检查

**错误示例**：
```solidity
pragma solidity ^0.7.0;  // ❌ 版本太低
pragma solidity ^0.9.0;  // ❌ 版本太高
```

**正确示例**：
```solidity
pragma solidity ^0.8.20; // ✅
pragma solidity 0.8.24;  // ✅
```

---

#### 2. `func-visibility`
```json
"func-visibility": ["warn", { "ignoreConstructors": true }]
```
- **级别**：`warn`（警告）
- **作用**：函数必须显式声明可见性
- **配置**：忽略 constructor（构造函数可省略 public）
- **原因**：
  - 明确函数的访问权限
  - 防止意外暴露内部函数
  - 提高代码可读性

**可见性类型**：
- `public` - 任何人都可以调用（内部和外部）
- `external` - 只能从外部调用（节省 gas）
- `internal` - 只能从合约内部或继承合约调用
- `private` - 只能从当前合约调用

**错误示例**：
```solidity
function transfer(address to, uint amount) {  // ❌ 缺少可见性
    // ...
}
```

**正确示例**：
```solidity
function transfer(address to, uint amount) public {  // ✅
    // ...
}

function _updateBalance(address account) internal {  // ✅
    // ...
}
```

---

#### 3. `no-console`
```json
"no-console": "warn"
```
- **级别**：`warn`（警告）
- **作用**：检测 `hardhat/console.sol` 的导入和使用
- **原因**：
  - console 是调试工具，不应该出现在生产代码
  - 增加合约字节码大小
  - 浪费部署 gas

**错误示例**：
```solidity
import "hardhat/console.sol";  // ⚠️ 开发阶段可以，部署前移除

function test() public {
    console.log("Debug message");
}
```

**部署前处理**：
- 移除所有 `import "hardhat/console.sol"`
- 移除所有 `console.log()` 调用
- 或将规则级别改为 `"error"` 强制检查

---

#### 4. `no-empty-blocks`
```json
"no-empty-blocks": "warn"
```
- **级别**：`warn`（警告）
- **作用**：检测空的代码块
- **原因**：
  - 空代码块通常是未完成的代码
  - 或者忘记删除的无用代码
  - 降低代码质量

**错误示例**：
```solidity
function doSomething() public {
    // ⚠️ 空函数体
}

if (condition) {
    // ⚠️ 空 if 块
}
```

**正确示例**：
```solidity
// 方案1：实现功能
function doSomething() public {
    balance += 1;
}

// 方案2：如果确实需要空函数（如占位符）
function doSomething() public {
    // solhint-disable-next-line no-empty-blocks
    // Placeholder for future implementation
}

// 方案3：使用 virtual 表示待子合约实现
function doSomething() public virtual {
    // Implemented in derived contracts
}
```

---

#### 5. `no-unused-vars`
```json
"no-unused-vars": "warn"
```
- **级别**：`warn`（警告）
- **作用**：检测未使用的变量
- **原因**：
  - 清理无用代码
  - 减少合约大小
  - 提高代码可读性

**错误示例**：
```solidity
function calculate() public {
    uint256 result;      // ⚠️ 声明了但未使用
    uint256 temp = 10;   // ⚠️ 赋值了但未使用
    return 5;
}
```

**正确示例**：
```solidity
function calculate() public returns (uint256) {
    uint256 result = 10;
    result += 5;
    return result;  // ✅ 变量被使用了
}

// 如果确实需要占位变量
function getData() public returns (uint256, uint256) {
    (, uint256 value) = someFunction();  // ✅ 用逗号忽略不需要的返回值
    return value;
}
```

---

#### 6. `avoid-low-level-calls`
```json
"avoid-low-level-calls": "warn"
```
- **级别**：`warn`（警告）
- **作用**：检测 `.call()`, `.delegatecall()`, `.staticcall()` 的使用
- **原因**：
  - 绕过类型检查，容易出错
  - 不会自动回滚（需要手动检查 success）
  - 安全风险较高

**危险示例**：
```solidity
// ⚠️ 低级调用，没有类型检查
(bool success, ) = target.call(abi.encodeWithSignature("transfer(address,uint256)", to, amount));
require(success, "Call failed");
```

**推荐方案**：
```solidity
// ✅ 使用接口调用（有类型检查）
IERC20(token).transfer(to, amount);

// 或者明确告诉 linter 你知道风险
// solhint-disable-next-line avoid-low-level-calls
(bool success, ) = target.call(data);
require(success, "Call failed");
```

---

#### 7. `reason-string`
```json
"reason-string": ["warn", { "maxLength": 64 }]
```
- **级别**：`warn`（警告）
- **作用**：`require` 和 `revert` 必须提供错误信息
- **配置**：错误信息最长 64 个字符
- **原因**：
  - 帮助调试（知道为什么失败）
  - 提升用户体验
  - 但太长会浪费 gas

**错误示例**：
```solidity
require(balance > 0);  // ❌ 没有错误信息
revert();              // ❌ 没有错误信息

require(msg.sender == owner, "This is a very long error message that exceeds the maximum length limit and wastes gas");  // ⚠️ 超过 64 字符
```

**正确示例**：
```solidity
require(balance > 0, "Insufficient balance");  // ✅ 清晰简洁
revert("Not authorized");                      // ✅

// 或使用自定义错误（Solidity 0.8.4+，更省 gas）
error InsufficientBalance();
if (balance == 0) revert InsufficientBalance();  // ✅
```

## 📊 规则级别说明

| 级别 | 含义 | 效果 | 使用场景 |
|------|------|------|---------|
| `"off"` / `0` | 关闭 | 不检查 | 不需要的规则 |
| `"warn"` / `1` | 警告 | 显示警告，不阻止编译 | 开发阶段，允许临时违反 |
| `"error"` / `2` | 错误 | 报错，lint 失败 | 必须遵守的规则（如部署前） |

### 常用规则推荐

#### 命名规范
```json
"contract-name-camelcase": "warn",    // 合约名 PascalCase
"func-name-mixedcase": "warn",        // 函数名 camelCase
"var-name-mixedcase": "warn",         // 变量名 camelCase
"const-name-snakecase": "warn",       // 常量 UPPER_SNAKE_CASE

```

#### 安全规则
```json
"avoid-suicide": "error",             // 禁用 selfdestruct
"avoid-sha3": "error",                // 用 keccak256 代替 sha3
"avoid-throw": "error",               // 用 revert 代替 throw
"reentrancy": "warn",                 // 重入检查
"state-visibility": "warn"            // 状态变量可见性
```

#### Gas 优化
```json
"func-param-data-location": "warn",   // 外部函数用 calldata
"immutable-vars-naming": "warn"       // 优先使用 immutable
```

## ✅ 检查清单

写完合约后，确保：

- [ ] 没有 `console.sol` import（或已标记为临时）
- [ ] 所有函数都有可见性声明
- [ ] 没有空的代码块
- [ ] 没有未使用的变量
- [ ] 所有 `require` 都有错误信息
- [ ] 没有低级调用（或已明确标注原因）
- [ ] 编译器版本正确

## 💡 忽略规则的方法

### 忽略下一行
```solidity
// solhint-disable-next-line no-console
import "hardhat/console.sol";
```

### 忽略多行
```solidity
/* solhint-disable no-console */
import "hardhat/console.sol";
console.log("Debug");
/* solhint-enable no-console */
```

### 忽略整个文件
```solidity
/* solhint-disable */
pragma solidity ^0.8.20;
// 整个文件不检查
```

### 忽略特定规则
```solidity
/* solhint-disable no-console, no-empty-blocks */
// 只忽略这两个规则
```
