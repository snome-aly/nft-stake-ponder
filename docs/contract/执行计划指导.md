### Week 2: NFTStakingPool 核心质押逻辑

#### Step 2.1: 创建 NFTStakingPool 合约（3天）

```bash
touch packages/hardhat/contracts/staking/NFTStakingPool.sol
```

**核心功能**（需求文档第 322-445 行）：
1. `stake(uint256 tokenId)` - 质押NFT
2. `unstake(uint256 tokenId)` - 取回NFT + 领取奖励
3. `claimRewards(uint256 tokenId)` - 单独领取奖励
4. `calculateRewards(uint256 tokenId)` - 计算待领取奖励

**关键点**：
- 使用 `ERC721Holder` 接收NFT
- 使用 `ReentrancyGuard` 防重入
- 奖励公式：`baseRewardPerSecond * duration * rarityMultiplier / 100`

#### Step 2.2: 创建测试（2天）

```bash
touch packages/hardhat/test/NFTStakingPool.test.ts
```

**测试场景**：
```typescript

  it("用户可以取回NFT");
  it("非owner无法取回他人的NFT");
  it("奖励随时间线性增长");
});
```

#### Step 2.3: 更新部署脚本（1天）

// 在 NFTStakingPool 合约中添加
address public achievementTrigger;

function setAchievementTrigger(address _trigger) external onlyOwner {
    achievementTrigger = _trigger;

if (achievementTrigger != address(0)) {
    IAchievementTrigger(achievementTrigger).onStake(msg.sender, tokenId);
}

## 🔧 常见问题解决

### 问题1: 合约编译错误
```bash
# 清理缓存重新编译
yarn hardhat:clean
yarn hardhat:compile
```

### 问题2: 测试失败
```bash
# 查看详细错误信息
yarn hardhat:test --verbose

# 只运行特定测试
yarn hardhat:test --grep "测试名称"
```

### 问题3: 前端无法读取合约
- 检查 `deployedContracts.ts` 是否已更新
- 确保本地链正在运行
- 检查钱包是否连接

### 问题4: Ponder 不索引事件
```bash
# 重启 Ponder
# 删除 .ponder 目录重新索引
rm -rf packages/ponder/.ponder
yarn ponder:dev
```
