> 直观理解 Hardhat Deploy 的工作流程

## 📊 完整部署流程图

```
开发者执行 yarn deploy
         ↓
  ┌──────────────────┐
  │ Hardhat Deploy   │
  │   启动           │
  └────────┬─────────┘
           ↓
  ┌────────────────────────────┐
  │ 读取 deploy/ 目录          │
  │ 按文件名排序：             │
  │ 00_xxx.ts → 01_xxx.ts ...  │
  └────────┬───────────────────┘
           ↓
    对每个部署脚本：
           ↓
  ┌─────────────────────────────────┐
  │ 1. 检查依赖                      │
  │    dependencies: ["Token"]       │
  │    确保依赖的合约已部署           │
  └────────┬────────────────────────┘
           ↓
  ┌─────────────────────────────────────┐
  │ 2. 执行 deploy() 函数               │
  └────────┬────────────────────────────┘
           ↓
  ┌──────────────────────────────────────┐
  │ 3. 检查部署记录                      │
  │    deployments/{network}/Contract.json│
  └────────┬─────────────────────────────┘
           ↓
    ┌──────┴──────┐
    │ 记录存在？  │
    └──┬──────┬───┘
       否      是
       ↓       ↓
    部署   ┌─────────────────────┐
    合约   │ 对比关键参数：      │
       ↓   │ • args (构造参数)   │
    保存   │ • solcInputHash    │
    记录   │   (代码哈希)        │
       ↓   └────┬──────┬─────────┘
       │        ↓      ↓
       │      相同    不同
       │       ↓      ↓
       │    跳过    重新
       │    部署    部署
       │       ↓      ↓
       └───────┴──────┘
              ↓
  ┌─────────────────────────────┐
  │ 4. 继续执行 deploy() 之后   │
  │    的代码（初始化等）        │
  │    ⚠️ 这部分总是执行！      │
  └────────┬────────────────────┘
           ↓
  ┌─────────────────────────────┐
  │ 5. 执行下一个部署脚本        │
  └────────┬────────────────────┘
           ↓
      所有脚本执行完成
           ↓
  ┌─────────────────────────────┐
  │ 自动生成前端类型文件：       │
  │ deployedContracts.ts        │
  └─────────────────────────────┘
```

## 🎯 单文件 vs 多文件执行流程

### 单文件策略

```
00_deploy_stakable_nft.ts
         ↓
  ┌─────────────────┐
  │ 1. deploy()     │ ✅ 自动幂等
  └────────┬────────┘
           ↓
  ┌─────────────────┐
  │ 2. 设置稀有度池  │ ⚠️ 需手动检查
  └────────┬────────┘
           ↓
  ┌─────────────────┐
  │ 3. 配置角色     │ ⚠️ 需手动检查
  └────────┬────────┘
           ↓
         完成

优点：一次执行完成
缺点：不能选择性执行
```

### 多文件策略

```
  00_deploy_nft.ts          01_setup_rarity.ts      02_setup_roles.ts
         ↓                          ↓                       ↓
  ┌─────────────┐          ┌──────────────┐        ┌──────────────┐
  │ deploy()    │ ─依赖─→  │ 设置稀有度池  │ ─依赖→ │ 配置角色     │
  └─────────────┘          └──────────────┘        └──────────────┘
   ✅ 自动幂等              ⚠️ 手动检查             ⚠️ 手动检查

优点：
• 关注点分离
• 可选择性执行：yarn deploy --tags SetupRoles
• 易于维护

缺点：
• 文件较多
• 需管理依赖
```

## 📂 目录结构与执行关系

```
项目结构                     执行时机                用途

deploy/
├── 00_deploy_nft.ts  ──→  yarn deploy        ──→  部署合约
├── 01_setup_rarity.ts ──→  yarn deploy        ──→  一次性初始化
└── 02_setup_roles.ts  ──→  yarn deploy        ──→  一次性配置

scripts/
├── management/
│   ├── grantRole.ts   ──→  手动执行           ──→  授予新角色
│   └── updateMultipliers.ts → 手动执行        ──→  更新参数
└── operations/
    ├── pause.ts       ──→  紧急时手动执行     ──→  暂停合约
    └── withdraw.ts    ──→  定期手动执行       ──→  提现

deployments/
└── localhost/
    ├── .chainId       ──→  自动生成           ──→  链 ID 验证
    └── StakableNFT.json → 自动生成           ──→  部署状态记录
```

## 🔁 幂等性问题示意图

### ❌ 问题场景

```
第一次执行 yarn deploy
    ↓
┌─────────────────────────┐
│ deploy("NFT")           │ ✅ 部署成功
├─────────────────────────┤
│ nft.setRarityPool()     │ ✅ 设置成功
├─────────────────────────┤
│ nft.grantRole()         │ ✅ 授予成功
└─────────────────────────┘

第二次执行 yarn deploy（无修改）
    ↓
┌─────────────────────────┐
│ deploy("NFT")           │ ⏭️  跳过（幂等）
├─────────────────────────┤
│ nft.setRarityPool()     │ ❌ 报错（已设置）
├─────────────────────────┤
│ nft.grantRole()         │ 💸 浪费 gas
└─────────────────────────┘
```

### ✅ 解决方案

```
任意次执行 yarn deploy
    ↓
┌──────────────────────────────┐
│ deploy("NFT")                │ ⏭️  自动跳过
├──────────────────────────────┤
│ if (!rarityPoolSet) {        │ 🔍 检查状态
│   nft.setRarityPool()        │ ⏭️  已设置，跳过
│ }                             │
├──────────────────────────────┤
│ if (!hasRole) {              │ 🔍 检查状态
│   nft.grantRole()            │ ⏭️  已有，跳过
│ }                             │
└──────────────────────────────┘
         ↓
    ✅ 安全可靠
```

## 🎬 实际操作时间线

```
时间轴                操作                          状态变化

T0    │ 编写合约 StakableNFT.sol
      │
T1    │ 编写部署脚本
      │ deploy/00_deploy_stakable_nft.ts
      │
T2    │ 首次部署                          deployments/localhost/
      │ $ yarn deploy                      ├── .chainId ✅
      │                                    └── StakableNFT.json ✅
      │
T3    │ 修改合约代码
      │ 添加新函数
      │
T4    │ 再次部署                          deployments/localhost/
      │ $ yarn deploy                      └── StakableNFT.json 🔄 更新
      │ 检测到代码变化 → 重新部署           （新地址）
      │
T5    │ 无修改，重复部署                   deployments/localhost/
      │ $ yarn deploy                      └── StakableNFT.json ⏭️  不变
      │ ✓ 重用现有部署
      │
T6    │ 运营中授予新角色                   不影响 deployments/
      │ $ yarn hardhat run                 （链上状态变化）
      │   scripts/management/grantRole.ts
```
